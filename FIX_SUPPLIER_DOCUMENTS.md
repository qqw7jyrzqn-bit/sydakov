# Исправление доступа поставщиков к документам

## Проблема
Поставщики не могли скачивать технические задания и другие документы.

## Решение
Внесены следующие изменения в [documents/views.py](c:\Users\Никита\Desktop\курсач 3 курс типу\12\documents\views.py):

### 1. Функция `generate_report()`:
- **Было:** Декоратор `@can_generate_documents_required` блокировал доступ всем, кроме `pto_engineer` и `head_pto`
- **Стало:** 
  - Поставщики **могут скачивать** существующие документы
  - Поставщики **НЕ могут создавать** новые документы
  - Если документ не существует, поставщик получит сообщение: *"Документ ещё не создан. Обратитесь к инженеру ПТО."*

### 2. Функция `download_document()`:
- **Добавлена проверка доступа:** Пользователь должен иметь доступ к скважине документа
- Использует `get_accessible_wells()` для проверки прав

### 3. Шаблон `well_detail.html`:
- **Изменён текст кнопки:**
  - Для ПТО: "Сформировать документ"
  - Для поставщиков: "Скачать документ"
- Dropdown с документами теперь виден всем пользователям, у которых есть доступ к скважине

## Как это работает

### Для инженера ПТО (pto_engineer, head_pto):
1. Видит кнопку **"Сформировать документ"**
2. При клике может:
   - Создать новый документ (если его нет)
   - Скачать существующий документ

### Для поставщика (supplier):
1. Видит кнопку **"Скачать документ"**
2. При клике может:
   - ✅ Скачать существующий документ
   - ❌ НЕ может создать новый документ

## Код изменений

### documents/views.py (строка 57):
```python
@login_required  # Убран @can_generate_documents_required
def generate_report(request, well_id):
    # Проверка: если документа нет и пользователь - поставщик
    if not document and request.user.role == 'supplier':
        messages.error(request, 'Документ ещё не создан...')
        return redirect('wells:well_detail', pk=well_id)
```

### documents/views.py (строка 130):
```python
@login_required
def download_document(request, pk):
    # Проверка доступа к скважине документа
    accessible_wells = request.user.get_accessible_wells()
    if document.well not in accessible_wells:
        messages.error(request, 'У вас нет доступа к этому документу.')
        return redirect('documents:document_list')
```

### templates/wells/well_detail.html (строка 199):
```html
<button>
    {% if can_generate_docs %}Сформировать{% else %}Скачать{% endif %} документ
</button>
```
