# Пошаговая инструкция по запуску проекта

## Быстрый старт (для проверяющего преподавателя)

### 1. Убедитесь, что установлен Python 3.11+
```powershell
python --version
```

### 2. Перейдите в директорию проекта
```powershell
cd "c:\Users\Никита\Desktop\курсач 3 курс типу\12"
```

### 3. Создайте виртуальное окружение
```powershell
python -m venv venv
```

### 4. Активируйте виртуальное окружение
```powershell
.\venv\Scripts\Activate.ps1
```

Если появляется ошибка безопасности PowerShell, выполните:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 5. Установите зависимости
```powershell
pip install -r requirements.txt
```

### 6. Примените миграции базы данных
```powershell
python manage.py migrate
```

### 7. Создайте демо-данные
```powershell
python manage.py demo_data
```

Эта команда создаст:
- 5 пользователей с разными ролями
- 3 скважины в разных статусах
- Тестовые документы и уведомления

### 8. Запустите сервер
```powershell
python manage.py runserver
```

### 9. Откройте браузер
Перейдите по адресу: http://127.0.0.1:8000/

## Проверка функционала

### 1. Создание скважины (Инженер ПТО)
- Войдите как `pto_engineer`
- Нажмите "Новая скважина"
- Заполните форму и сохраните

### 2. Отправка на согласование
- Откройте созданную скважину
- Нажмите "Отправить на согласование"

### 3. Согласование (Начальник ПТО)
- Войдите как `head_pto`
- Откройте скважину со статусом "На согласовании"
- Оставьте комментарий и нажмите "Согласовать"

### 4. Генерация отчёта
- На странице любой скважины нажмите "Сформировать отчёт"
- Скачается Word-файл (.docx) с данными скважины

### 5. Экспорт в Excel
- На главной странице нажмите "Экспорт"
- Скачается Excel-файл со списком всех скважин

### 6. Уведомления
- Обратите внимание на иконку колокольчика в навигации
- При изменении статусов скважин создаются уведомления

### 7. Портал поставщика
- Войдите как `supplier`
- Вы увидите только технические задания (readonly)

## Возможные проблемы и решения

### Ошибка "No module named 'django'"
Убедитесь, что виртуальное окружение активировано:
```powershell
.\venv\Scripts\Activate.ps1
```

### Ошибка миграций
Пересоздайте базу данных:
```powershell
del db.sqlite3
python manage.py migrate
python manage.py demo_data
```

### Статические файлы не загружаются
В файле `.env` проверьте, что `DEBUG=True`

### Email не отправляются
По умолчанию используется консольный бэкенд - письма выводятся в терминал, где запущен сервер
