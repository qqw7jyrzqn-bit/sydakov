## API Endpoints

### Мессенджер:

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/accounts/chat/users/?search=query` | Поиск пользователей |
| GET | `/accounts/chat/conversations/` | Список диалогов |
| GET | `/accounts/chat/messages/<user_id>/` | Сообщения с пользователем |
| POST | `/accounts/chat/send/` | Отправить сообщение |
| GET | `/accounts/chat/unread/` | Количество непрочитанных |

## Использование

### Как начать чат:

1. **Откройте виджет** - кликните на круглую кнопку справа внизу
2. **Найдите пользователя** - введите имя в поле поиска
3. **Выберите собеседника** - кликните на пользователя из списка
4. **Отправьте сообщение** - напишите текст и нажмите Enter или кнопку отправки

### Навигация в мессенджере:

- **Кнопка "←"** - вернуться к списку диалогов
- **Кнопка "×"** - закрыть виджет
- **Круглая кнопка** - открыть виджет снова

### Непрочитанные сообщения:

- Красный бейдж на кнопке виджета
- Синий бейдж на диалоге в списке
- Автоматически отмечаются как прочитанные при открытии

## Модель ChatMessage

### Поля:

```python
sender          # Отправитель (ForeignKey → User)
recipient       # Получатель (ForeignKey → User)
message         # Текст сообщения
created_at      # Дата отправки
is_read         # Прочитано (True/False)
read_at         # Дата прочтения
```

### Методы:

```python
mark_as_read()                          # Отметить как прочитанное
get_conversation(user1, user2)          # Переписка между двумя пользователями
get_unread_count(user)                  # Количество непрочитанных
get_conversations_list(user)            # Список всех диалогов
```

### Административная панель:

- URL: `/admin/accounts/chatmessage/`
- Фильтры: прочитано, дата
- Поиск: по отправителю, получателю, тексту

## Email-уведомления

### Настройка:

Добавьте в `.env`:

```env
# Консольный вывод (для разработки)
EMAIL_BACKEND=django.core.mail.backends.console.EmailBackend

# Или реальная почта (Gmail)
EMAIL_BACKEND=django.core.mail.backends.smtp.EmailBackend
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password
```

### Когда отправляются:

1. **Новое сообщение в чате** → Получатель получает email
2. **Создание уведомления** → Пользователь получает email
3. **Автоматически через signals** → post_save на модели Notification

### Формат письма:

**Тема:** `[Система ПТО] Заголовок уведомления`

**Содержание:**
- Приветствие с именем пользователя
- Заголовок уведомления
- Текст сообщения
- Дата и время
- Кнопка "Перейти в систему"
- Footer

## Особенности реализации

### JavaScript (AJAX):

- Используется Fetch API
- CSRF токены из cookies
- Автообновление через setInterval
- Плавные анимации открытия/закрытия

### CSS:

- Gradient background (фиолетовый)
- Flex layout для адаптивности
- Custom scrollbar стили
- Bubble дизайн для сообщений
- Разные цвета для "мои" и "чужие" сообщения

### Backend:

- Django Signals для email
- Индексы БД для производительности
- Защита @login_required на все endpoints
- JSON responses для AJAX

### Безопасность:

- CSRF защита на POST запросах
- Фильтрация по правам доступа
- Нельзя отправить сообщение самому себе
- Проверка существования получателя
