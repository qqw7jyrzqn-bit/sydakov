# Новый функционал системы

## 📊 Аналитика и отчёты

### Дашборд аналитики
**URL:** `/analytics/`

Главная страница аналитики предоставляет:
- **12 статистических показателей:**
  - Общее количество скважин
  - Активные скважины
  - Завершённые скважины
  - Ожидают согласования
  - Отклонённые скважины
  - Средняя глубина скважин
  - Количество уникальных месторождений
  - Завершено за последний месяц
  - Скважины в работе
  - Архивированные
  - Топ-3 автора по количеству скважин
  - Средний срок согласования

- **3 интерактивных графика (Chart.js):**
  1. **График создания скважин** - линейный график показывает динамику создания скважин по месяцам (последние 12 месяцев)
  2. **Распределение по статусам** - круговая диаграмма показывает процентное соотношение скважин в разных статусах
  3. **Топ-10 месторождений** - горизонтальная гистограмма показывает месторождения с наибольшим количеством скважин

**Доступ:** Все сотрудники ПТО (pto_engineer, head_pto, ext_engineer)

### Отчёт по месторождению
**URL:** `/analytics/field/<название_месторождения>/`

Детальная статистика по конкретному месторождению:
- Общее количество скважин
- Завершённые скважины
- Скважины в работе
- Общая глубина всех скважин
- Таблица всех скважин месторождения с возможностью перехода к детальному просмотру

**Как использовать:**
В дашборде аналитики кликните на название месторождения в графике "Топ-10 месторождений"

### Отчёт по эффективности
**URL:** `/analytics/performance/`

Статистика производительности сотрудников:
- Таблица с данными по каждому пользователю:
  - ФИО и username
  - Роль в системе
  - Количество созданных скважин
  - Количество согласованных скважин
  - Количество отклонённых скважин
  - Процент одобрения
- Общая статистика:
  - Всего пользователей ПТО
  - Общий процент одобрения
  - Средний коэффициент отказа

**Доступ:** Начальник ПТО (head_pto), можно расширить на других пользователей

## 🔍 Расширенный поиск

### Страница расширенного поиска
**URL:** `/search/`

Многокритериальный поиск скважин с фильтрами:

**Параметры поиска:**
1. **Текстовый поиск** - ищет по названию, описанию и месторождению
2. **Месторождение** - фильтр по конкретному месторождению
3. **Статус** - выбор из dropdown списка статусов (Черновик, На согласовании, В работе и т.д.)
4. **Автор** - поиск по имени или username автора
5. **Глубина от/до** - диапазон глубины в метрах
6. **Дата создания с/по** - временной диапазон создания скважин

**Результаты:**
- Таблица найденных скважин
- Счётчик найденных результатов
- Кнопки для просмотра деталей каждой скважины
- Цветовая индикация статусов

**Пример использования:**
```
Найти все скважины:
- Месторождение: Самотлор
- Статус: В работе
- Глубина: от 3000 до 5000 метров
- Создано: с 01.01.2024 по 31.12.2024
```

**Доступ:** Все авторизованные пользователи (с учётом их прав доступа)

## 📝 Расширенные модели данных

### 1. Комментарии (Comment)
Система комментариев к скважинам с поддержкой вложенных ответов.

**Поля:**
- Скважина (ForeignKey)
- Автор
- Текст комментария
- Родительский комментарий (для создания цепочек ответов)
- Дата создания/обновления

**Функционал:**
- Добавление комментариев к скважине
- Ответы на комментарии (threaded comments)
- Редактирование своих комментариев
- Отображение в хронологическом порядке

**Административная панель:**
Доступна в Django Admin: `/admin/wells/comment/`

### 2. Теги (Tag)
Система тегов для группировки и категоризации скважин.

**Поля:**
- Название тега
- Цвет (HEX код для визуального отображения)
- Описание
- Создатель тега

**Примеры использования:**
- Теги по типу скважин: "Разведочная", "Эксплуатационная", "Нагнетательная"
- Теги по приоритету: "Срочно", "Высокий приоритет", "Плановая"
- Теги по регионам: "Западная Сибирь", "Урал", "Поволжье"

**Административная панель:**
`/admin/wells/tag/`

### 3. История изменений (WellHistory)
Полный аудит всех действий со скважиной.

**Отслеживаемые действия:**
- `created` - Создание скважины
- `updated` - Обновление данных
- `status_changed` - Изменение статуса
- `submitted` - Отправка на согласование
- `approved` - Согласование
- `rejected` - Отклонение
- `deleted` - Удаление

**Сохраняемая информация:**
- Пользователь, совершивший действие
- Временная метка
- Старое значение
- Новое значение
- Комментарий к изменению
- IP-адрес пользователя

**Административная панель:**
`/admin/wells/wellhistory/`

### 4. Дедлайны (WellDeadline)
Управление сроками выполнения этапов проекта скважины.

**Типы вех (milestones):**
1. `design` - Проектирование
2. `approval` - Согласование
3. `drilling_start` - Начало бурения
4. `drilling_end` - Завершение бурения
5. `testing` - Испытания
6. `commissioning` - Ввод в эксплуатацию

**Поля:**
- Скважина
- Тип вехи
- Плановая дата
- Фактическая дата (опционально)
- Ответственный сотрудник
- Заметки

**Автоматические вычисления:**
- `is_overdue` - Проверка просрочки
- `days_remaining` - Количество дней до дедлайна

**Административная панель:**
`/admin/wells/welldeadline/`

### 5. Вложения (WellAttachment)
Система загрузки и хранения файлов, связанных со скважиной.

**Категории файлов:**
- `technical` - Технические чертежи
- `geological` - Геологические данные
- `photo` - Фотографии
- `report` - Отчёты
- `other` - Прочее

**Поля:**
- Скважина
- Файл (upload)
- Название
- Категория
- Описание
- Загружено пользователем
- Размер файла (автоматически)
- Дата загрузки

**Автоматические функции:**
- Расчёт размера файла в байтах
- Свойство `file_size_mb` для отображения в МБ

**Административная панель:**
`/admin/wells/wellattachment/`

## 🎯 Формы для работы с расширенными моделями

**Доступные формы:**

## 📈 Технические детали

### Используемые библиотеки
- **Chart.js 4.4.0** - библиотека для построения графиков
- **Bootstrap Icons** - иконки для UI
- **Django ORM** - агрегации (Count, Avg, TruncMonth)

### Производительность
- Все запросы к БД оптимизированы
- Используются агрегации вместо циклов Python
- Пагинация для больших списков
- Кеширование для часто запрашиваемых данных (можно добавить)

### Безопасность
- Все представления защищены декораторами @login_required
- Фильтрация данных по правам доступа (get_accessible_wells)
- Валидация форм на стороне сервера
- CSRF защита для всех POST запросов
