# Система инженера производственно-технического отдела (ПТО)

## Описание проекта

Информационная система для автоматизации деятельности инженера ПТО при проектировании буровых скважин, контроле их строительства и технадзоре за бурением.

**Курсовая работа 3 курс**  
**Стек:** Django 4.2 LTS, Python 3.11, Bootstrap 5.3, SQLite

## Возможности системы

### Управление скважинами
- Создание, редактирование и удаление скважин
- Маршрут согласования с переходами статусов:
  - Черновик → Отправлено на согласование → Согласовано НПТО → Утверждено → В работе → Бурение завершено → Архив
- Timeline истории согласований
- Фильтрация и поиск скважин
- Экспорт списка скважин в Excel

### Документы и отчёты
- Загрузка технических заданий и отчётов
- Генерация Word-отчётов по скважине (python-docx)
- Скачивание документов

### Система уведомлений
- In-app уведомления в навигационной панели
- Email-уведомления о смене статусов
- Уведомления при упоминании в комментариях

### Разграничение ролей
1. **Инженер ПТО** - полный CRUD по скважинам, техзаданиям, отчётам
2. **Начальник ПТО** - согласование/утверждение, дашборд по отделу
3. **Технические специалисты** - readonly + комментарии к скважинам
4. **Подрядчики/поставщики** - отдельный портал с readonly доступом к ТЗ

### Интерфейс
- Современный дизайн в нефтегазовой тематике (тёмно-синий)
- Адаптивная вёрстка Bootstrap 5.3
- Иконки Bootstrap Icons
- Toast-уведомления

## Установка и запуск

```bash

```

### 2. Создание виртуального окружения

```powershell
python -m venv venv

```

### 3. Установка зависимостей

```powershell

```

### 4. Настройка переменных окружения

Скопируйте `.env.example` в `.env` и настройте при необходимости:

```powershell

```

По умолчанию используется SQLite и консольный бэкенд для email.

### 5. Применение миграций

```powershell
python manage.py migrate
```

### 6. Создание демо-данных

```powershell
python manage.py demo_data
```

Эта команда создаст:
- 5 пользователей с разными ролями
- 3 скважины в разных статусах
- Тестовые документы и уведомления

### 7. Запуск сервера

```powershell
python manage.py runserver
```

Откройте браузер: http://127.0.0.1:8000/

## Генерация отчётов

Кнопка **"Сформировать отчёт"** на странице скважины создаёт Word-документ (.docx) с:
- Основными данными скважины
- Таблицей истории согласований
- Комментариями к согласованиям

Файл сохраняется в `media/generated_reports/` и автоматически скачивается.

## Экспорт в Excel

Кнопка **"Экспорт"** на дашборде создаёт Excel-файл со списком всех скважин.

## Технические детали

### База данных
По умолчанию используется SQLite (`db.sqlite3`). Для production рекомендуется PostgreSQL.

### Email
По умолчанию используется `console.EmailBackend` (письма выводятся в консоль).  
Для реальной отправки настройте SMTP в `.env`.

### Статические файлы
В режиме разработки Django автоматически обслуживает статику.  
Для production выполните:

```powershell
python manage.py collectstatic
```

### Права доступа
Система использует Django permissions и кастомные миксины:
- `PTOEngineerRequiredMixin` - доступ для инженера ПТО
- `HeadPTORequiredMixin` - доступ для начальника ПТО
- `SupplierRequiredMixin` - доступ для поставщиков
- `PTOStaffRequiredMixin` - доступ для сотрудников ПТО

## Troubleshooting

### Ошибка импорта модулей
Убедитесь, что виртуальное окружение активировано:
```powershell
.\venv\Scripts\Activate.ps1
```

### Ошибки миграций
Пересоздайте базу данных:
```powershell
del db.sqlite3
python manage.py migrate
python manage.py demo_data
```

### Статические файлы не загружаются
Проверьте, что `DEBUG=True` в `.env` или выполните:
```powershell
python manage.py collectstatic
```
