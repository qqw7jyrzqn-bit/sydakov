# 🧪 Руководство по тестированию разграничения доступа

## Цель тестирования
Проверить, что каждая роль имеет доступ только к разрешённым функциям и не может выполнять запрещённые действия.

---

## 📝 Тестовые пользователи

| Логин | Пароль | Роль | Описание |
|-------|--------|------|----------|
| `admin` | `admin` | Начальник ПТО | Полный доступ + админка |
| `pto_engineer` | `1234` | Инженер ПТО | Создание и управление скважинами |
| `head_pto` | `1234` | Начальник ПТО | Согласование проектов |
| `ext_engineer` | `1234` | Специалист | Просмотр и комментирование |
| `supplier` | `1234` | Поставщик | Ограниченный доступ |

---

## 🧩 Тест 1: Инженер ПТО

### ✅ Должно работать:
1. **Войти** в систему под `pto_engineer / 1234`
2. **Увидеть** кнопку "Создать скважину" в навбаре
3. **Создать** новую скважину:
   - Название: "Скважина №TEST-1"
   - Месторождение: "Тестовое"
   - Глубина: 2500м
   - Статус автоматически: "Черновик"
4. **Отредактировать** созданную скважину
5. **Отправить** скважину на согласование
6. **Сгенерировать** документ по скважине (любой тип)
7. **Просмотреть** все скважины в списке
8. **Экспортировать** список скважин в Excel

### ❌ НЕ должно работать:
1. **Согласовать** или **Отклонить** скважину (кнопки скрыты)
2. **Редактировать** чужие скважины (не автор)
3. **Удалить** утверждённую скважину
4. **Получить доступ** к `/admin/` (403 Forbidden)

### Проверка через прямые URL:
```
# Должна работать:
http://127.0.0.1:8080/wells/create/
http://127.0.0.1:8080/documents/generate/1/?type=report

# Должна блокировать:
http://127.0.0.1:8080/wells/1/approve/  → Ошибка "Только начальник ПТО..."
http://127.0.0.1:8080/admin/  → 403 Forbidden
```

## 🔒 Тест безопасности: Обход защиты

### Попытки обхода (все должны провалиться):

1. **Direct URL manipulation:**
   ```
   # Войти как ext_engineer, попытаться:
   http://127.0.0.1:8080/wells/create/  → Ошибка
   http://127.0.0.1:8080/wells/1/update/  → Ошибка
   http://127.0.0.1:8080/wells/1/approve/  → Ошибка
   ```

2. **POST requests без прав:**
   ```bash
   # Попытка создать скважину через cURL (без прав)
   curl -X POST http://127.0.0.1:8080/wells/create/ \
     -H "Cookie: sessionid=..." \
     -d "name=Hack&field=Test&depth=1000"
   # Должна вернуть: 403 Forbidden или редирект с ошибкой
   ```

3. **Session hijacking simulation:**
   - Войти как `ext_engineer`
   - Скопировать sessionid из cookies
   - Попытаться выполнить действия начальника ПТО
   - **Результат:** Все должно блокироваться

4. **Parameter tampering:**
   ```
   # Войти как pto_engineer, попытаться согласовать через изменение формы
   http://127.0.0.1:8080/wells/1/approve/
   # Должна вернуть: Ошибка "Только начальник ПТО может выполнить это действие"
   ```

## 🚨 Что делать при обнаружении проблем:

1. **UI показывает недоступную кнопку:**
   - Проверить условие `{% if can_xxx %}` в шаблоне
   - Убедиться, что переменная передаётся в context

2. **Прямой URL не блокируется:**
   - Проверить декоратор на view функции
   - Убедиться, что миксин применён к class-based view

3. **Пользователь видит чужие данные:**
   - Проверить фильтрацию через `get_accessible_wells()`
   - Убедиться, что QuerySet ограничен

4. **Ошибка 500 вместо 403:**
   - Проверить логи Django
   - Возможно, отсутствует импорт декоратора/миксина
